1. แนะนำ Lotus IDE1. Introduction

Lotus IDE คือสภาพแวดล้อมพัฒนาโปรแกรม (IDE) สำหรับบอร์ดหุ่นยนต์ตระกูล Lotus และบอร์ด Arduino ทั่วไป รองรับการเขียนโปรแกรมแบบบล็อก (Blockly) และแบบโค้ด C/C++ พร้อมระบบติดตั้งบอร์ดและปลั๊กอินผ่านอินเทอร์เน็ตในตัว

คุณสมบัติเด่น:

  • เขียนโปรแกรมแบบบล็อก (Blockly) ที่แปลงเป็นโค้ด Arduino C++ อัตโนมัติ
  • มี toolchain ของ Arduino-CLI ติดมาในตัว — ใช้งานออฟไลน์ได้ทันที
  • รองรับ AVR (Nano/Uno), SAM (Due), ESP32, ESP8266 และบอร์ดอื่น ๆ ผ่านระบบ Board Package
  • ระบบ Fast Compile สำหรับ ESP32 ทำให้คอมไพล์รอบที่ 2 เร็วขึ้น ~11 เท่า
  • จัดการ Plugin/Board ผ่าน catalog บน GitHub — เพิ่มบอร์ดใหม่ได้โดยไม่ต้องอัปเดต IDE
  • เผยแพร่ Plugin/Board ของคุณเองขึ้น GitHub ได้ภายใน IDE
  • เครื่องมือ ตรวจสุขภาพ ESP32 ช่วยวินิจฉัยและแก้ปัญหา toolchain อัตโนมัติ

Lotus IDE is a development environment for the Lotus robotics board family and general Arduino boards. It supports both Blockly (block-based) and C/C++ code programming, with built-in package and plugin management.

Key features:

  • Block-based programming (Blockly) that auto-generates Arduino C++ code
  • Bundled Arduino-CLI toolchain — works offline out of the box
  • Supports AVR (Nano/Uno), SAM (Due), ESP32, ESP8266, and more via the Board Package system
  • Fast Compile path for ESP32 — second build ~11× faster
  • Plugin / Board management via GitHub catalogs — add new boards without updating the IDE
  • Publish your own plugin or board to GitHub directly from the IDE
  • ESP32 Health Check tool — diagnoses and auto-fixes toolchain issues

2. การติดตั้ง2. Installation

Windows

  1. ดาวน์โหลดไฟล์ Lotus-IDE-Setup-x.y.z.exe จาก หน้า Releases
  2. ดับเบิลคลิกไฟล์ติดตั้ง — หาก Windows SmartScreen เตือน ให้กด More info → Run anyway
  3. อ่านและยอมรับ EULA (สัญญาอนุญาตการใช้งาน)
  4. เลือกตำแหน่งติดตั้ง (แนะนำ C:\Program Files\Lotus IDE)
  5. กด Install แล้วรอจนเสร็จ (ขนาดติดตั้ง ~1.5 GB เพราะมี toolchain ติดมาด้วย)
  1. Download Lotus-IDE-Setup-x.y.z.exe from the Releases page.
  2. Double-click the installer. If Windows SmartScreen warns, click More info → Run anyway.
  3. Read and accept the EULA.
  4. Choose an install location (default C:\Program Files\Lotus IDE).
  5. Click Install and wait (~1.5 GB — bundled toolchain).
⚠ Windows Defender
Windows Defender อาจสแกนไฟล์คอมไพล์ทำให้ช้ามาก แนะนำให้ใช้เมนู Lotus → ตรวจสุขภาพ ESP32 → Fix เพื่อเพิ่มโฟลเดอร์ Lotus IDE เข้า exclusion list (ต้องอนุมัติ UAC)
Defender may scan compiler artifacts and slow builds dramatically. Use Lotus → ตรวจสุขภาพ ESP32 → Fix to auto-add the Lotus IDE folder to Defender exclusions (requires UAC approval).

Linux (Ubuntu 20.04+)

มีให้เลือก 2 แบบ:

  • AppImage — ดาวน์โหลด Lotus-IDE-x.y.z.AppImage แล้วทำให้รันได้: 
    chmod +x Lotus-IDE-*.AppImage
./Lotus-IDE-*.AppImage
  • .deb — ติดตั้งด้วย: 
    sudo apt install ./lotus-ide_x.y.z_amd64.deb
    เวอร์ชัน .deb จะติดตั้ง udev rules ให้บอร์ดอัตโนมัติ (ไม่ต้อง sudo เวลาอัปโหลด)

Two formats:

  • AppImage — download Lotus-IDE-x.y.z.AppImage and make it executable: 
    chmod +x Lotus-IDE-*.AppImage
./Lotus-IDE-*.AppImage
  • .deb — install via: 
    sudo apt install ./lotus-ide_x.y.z_amd64.deb
    The .deb package installs udev rules automatically — no sudo needed for uploading.
macOS
ยังไม่รองรับอย่างเป็นทางการในเวอร์ชันนี้ (เลื่อนไปแผนงานถัดไป)
Not officially supported in this release — deferred to a future version.

3. เริ่มต้นใช้งาน3. First Run

  1. เปิดโปรแกรม — จะเห็นพื้นที่ Blockly ทางซ้าย และ pane โค้ดทางขวา
  2. เลือกบอร์ดจาก Tools → Select Board... (เช่น Arduino Nano, ESP32, Lotus DueBot)
  3. เลือกพอร์ตจาก dropdown ใน Toolbar (บอร์ดต้องเสียบ USB อยู่)
  4. ลากบล็อกจาก Toolbox มาประกอบใน workspace
  5. กด Verify (Ctrl+R) เพื่อคอมไพล์
  6. กด Upload (Ctrl+U) เพื่ออัปโหลดไปยังบอร์ด
  1. Launch the app — Blockly workspace on the left, code preview on the right.
  2. Pick a board from Tools → Select Board... (Arduino Nano, ESP32, Lotus DueBot, etc.).
  3. Pick a port from the dropdown in the toolbar (board must be plugged in via USB).
  4. Drag blocks from the Toolbox into the workspace.
  5. Hit Verify (Ctrl+R) to compile.
  6. Hit Upload (Ctrl+U) to flash the board.

6. การเขียนด้วย Blockly6. Blockly Editor

Toolbox ทางซ้ายแบ่งหมวดบล็อกตามฟังก์ชัน:

  • Logic / Loops / Math / Text / Variables / Functions — หมวดมาตรฐานของ Blockly
  • Input/Output / Time / Serial — บล็อกมาตรฐาน Arduino
  • หมวดเฉพาะบอร์ด — ติดตั้งผ่าน Plugin Setup จะแยกหมวดและสีของตัวเองตามชื่อปลั๊กอิน

โค้ดที่ Blockly สร้างจะแสดงสดที่ pane ขวา — สลับไปโหมด Code แล้วแก้เพิ่มได้

💡 เคล็ดลับ — บล็อก text Lotus IDE ปรับบล็อก text ให้สร้างสตริงแบบ "abc" (double-quote) แทน 'abc' (single-quote) เพื่อให้ตรงกับ Arduino String จริง

The Toolbox on the left groups blocks by category:

  • Logic / Loops / Math / Text / Variables / Functions — standard Blockly categories.
  • Input/Output / Time / Serial — standard Arduino blocks.
  • Board-specific categories — installed via Plugin Setup; each plugin gets its own category and color.

Generated code is shown live on the right — you can switch to Code Mode and continue editing.

💡 Tip — text block Lotus IDE overrides the stock text block to emit "abc" (double quotes) instead of 'abc' (single quotes), matching real Arduino String literals.

7. โหมด Code7. Code Mode

กด Ctrl+2 เพื่อสลับเป็นโหมดเขียน C/C++ โดยตรง — เหมาะกับผู้ใช้ขั้นกลางขึ้นไป

  • มี syntax highlight + auto-indent
  • กด Verify/Upload ใช้ได้ตามปกติ
  • ใช้คู่กับเมนู Lotus → Include Library เพื่อแทรก #include ได้
⚠ ระวัง เมื่ออยู่ในโหมด Code แล้วสลับกลับไป Blockly โค้ดที่แก้ไขเองจะถูกแทนที่ด้วยโค้ดที่ Blockly สร้างใหม่ — เก็บไฟล์เป็น .ino ไว้ก่อน

Press Ctrl+2 to switch to direct C/C++ editing — for intermediate users and up.

  • Syntax highlighting + auto-indent.
  • Verify / Upload work as usual.
  • Use Lotus → Include Library to insert #include directives.
⚠ Caution Switching from Code back to Blockly will overwrite your edits with newly generated code. Save as .ino first.

8. คอมไพล์ & อัปโหลด8. Compile & Upload

ขั้นตอน

  1. เลือกบอร์ดและพอร์ตให้ถูก
  2. กด Verify (Ctrl+R) — ทดลองคอมไพล์ก่อน
  3. ดู Console — ถ้ามี error จะมีสีแดง คลิกเพื่อ jump ไปบรรทัด (ในโหมด Code)
  4. เมื่อ Verify สำเร็จ กด Upload (Ctrl+U)

ESP32 Fast Compile

Lotus IDE มีระบบ Fast Compile สำหรับ ESP32 ที่จะ:

  • รอบแรก: คอมไพล์เต็มและบันทึก compile command ทุกตัวลง cache (%TEMP%/lotus_build/)
  • รอบถัดไป: รัน command เดิมซ้ำเฉพาะไฟล์ที่เปลี่ยน — เร็วขึ้น ~11.5 เท่า (จาก 213s เหลือ 18.5s)

หาก cache เพี้ยน (เช่น "header not found" ทั้งที่มีจริง) ให้ลบโฟลเดอร์ %TEMP%/lotus_build/ แล้วคอมไพล์ใหม่

Steps

  1. Pick board and port.
  2. Press Verify (Ctrl+R) to test-compile.
  3. Check the console — errors are red; click to jump to the line (in Code Mode).
  4. If verify succeeds, press Upload (Ctrl+U).

ESP32 Fast Compile

Lotus IDE includes a Fast Compile path for ESP32:

  • First build: full compile, captures every compile command in cache (%TEMP%/lotus_build/).
  • Subsequent builds: replays cached commands only for changed files — ~11.5× faster (213s → 18.5s).

If the cache gets confused (e.g. "header not found" even though it exists), delete %TEMP%/lotus_build/ and recompile.

9. Serial Monitor / Plotter9. Serial Monitor / Plotter

เปิดจาก Tools → Serial Monitor (Ctrl+Shift+M) หรือ Serial Plotter (Ctrl+Shift+L)

  • Baud rate เลือกได้ 9600 / 115200 ฯลฯ — ต้องตรงกับใน Serial.begin()
  • Line ending เลือกได้ None, \n, \r\n
  • Plotter รับค่าตัวเลขคั่นด้วย space หรือ comma — วาดเป็นกราฟอัตโนมัติ
  • เปิด Serial ค้างไว้แล้วอัปโหลดได้ — IDE จะปิดพอร์ตให้ชั่วคราว

Open via Tools → Serial Monitor (Ctrl+Shift+M) or Serial Plotter (Ctrl+Shift+L).

  • Baud rate: 9600 / 115200 / etc. — must match Serial.begin().
  • Line ending: None, \n, or \r\n.
  • Plotter: accepts space- or comma-separated numbers, auto-plots.
  • You can leave Serial open and upload — the IDE will release the port temporarily.

10. Boards Setup10. Boards Setup

เปิดจาก Lotus → Boards Setup...

แท็บหลัก:

  • Available — รายการบอร์ดทั้งหมดจาก catalog (default: jaturong-kamonlers/lotus-boards)
  • Installed — บอร์ดที่ติดตั้งแล้ว
  • Settings — แก้ URL ของ catalog เพิ่มเติมได้

กด Install ที่บอร์ดที่ต้องการ — IDE จะดาวน์โหลด toolchain + cores ให้อัตโนมัติ (ขนาด 50 MB – 1 GB ขึ้นกับบอร์ด)

💡 ESP32 ถ้า ESP32 core ติดตั้งไม่ครบ (ปกติเพราะถูก interrupt) ระบบจะตรวจจับและ "ติดตั้งซ่อม" ให้อัตโนมัติในการเปิดครั้งถัดไป — ตรวจสอบสถานะจาก ตรวจสุขภาพ ESP32 ได้

Open via Lotus → Boards Setup....

Main tabs:

  • Available — boards from the catalog (default: jaturong-kamonlers/lotus-boards).
  • Installed — installed boards.
  • Settings — add custom catalog URLs.

Click Install on a board — the IDE downloads its toolchain + cores (50 MB – 1 GB depending on the board).

💡 ESP32 If an ESP32 core install was interrupted, the IDE detects the half-installed state and repairs it on the next launch. Check via ESP32 Health Check.

11. Plugin Setup11. Plugin Setup

เปิดจาก Lotus → Plugin Setup...

Plugin = ชุดบล็อก Blockly + library + ตัวอย่าง สำหรับเซ็นเซอร์/มอเตอร์/บอร์ดเฉพาะ

  • catalog เริ่มต้น: jaturong-kamonlers/lotus-plugins มี 15 plugins ให้เลือก
  • ติดตั้งแล้วบล็อกจะปรากฏใน Toolbox ใต้หมวดแยก พร้อมสีของปลั๊กอินเอง
  • ถอนการติดตั้งได้ที่แท็บ Installed

Open via Lotus → Plugin Setup....

A plugin bundles Blockly blocks + Arduino library + examples for a sensor / motor / board.

  • Default catalog: jaturong-kamonlers/lotus-plugins (15 plugins).
  • Once installed, blocks appear under their own toolbox category with the plugin's color.
  • Uninstall from the Installed tab.

12. GitHub Integration12. GitHub Integration

เปิดจาก Lotus → GitHub — ใช้สำหรับเผยแพร่ plugin/board ของคุณเองขึ้น GitHub

  1. กด Sign in with GitHub — เปิด browser ให้ login (OAuth flow)
  2. เลือกปลั๊กอินหรือบอร์ดที่ติดตั้งแล้วและต้องการเผยแพร่
  3. กด Publish — IDE จะ:
    • zip ไฟล์ทั้งหมด
    • สร้าง GitHub repo (ถ้ายังไม่มี)
    • สร้าง release ใหม่ และ upload zip
💡 ใช้ catalog ของตัวเอง ในแท็บ Settings ของ Boards/Plugin Setup สามารถใส่ URL ของ catalog ที่ตนเองสร้างได้ — ทำให้ทีม/โรงเรียนของคุณเห็นบอร์ดที่จัดเตรียมไว้

Open via Lotus → GitHub — used to publish your own plugin or board to GitHub.

  1. Click Sign in with GitHub — opens the browser for OAuth login.
  2. Pick an installed plugin or board to publish.
  3. Click Publish — the IDE will:
    • Zip all files.
    • Create the GitHub repo if needed.
    • Create a new release and upload the zip asset.
💡 Custom catalogs In the Settings tab of Boards/Plugin Setup you can add a URL pointing to your own catalog — perfect for distributing curated boards to your team or classroom.

13. ตรวจสุขภาพ ESP3213. ESP32 Health Check

เปิดจาก Lotus → ตรวจสุขภาพ ESP32... เพื่อตรวจและแก้ปัญหา toolchain ของ ESP32 อัตโนมัติ

การตรวจสอบ:

  • ESP32 core ติดตั้งครบหรือไม่ — ตรวจที่ tools/ ไม่ใช่แค่ hardware/
  • gcc compiler — รองรับทั้ง esp-x32 (Xtensa) และ esp-rv32 (RISC-V) ของ core 3.x
  • Windows Defender exclusion — ถ้ายังไม่ได้เพิ่ม IDE เข้า exclusion list จะมีปุ่ม Fix ให้กด (เรียก UAC)
  • cache สถานะ — ตรวจ %TEMP%/lotus_build/ และเสนอให้ลบถ้าเพี้ยน
⚠ Defender slow build บน Windows ถ้าไม่เพิ่ม exclusion การคอมไพล์ครั้งแรกอาจช้ากว่าควรหลายเท่า — แนะนำให้กด Fix ครั้งเดียวตั้งแต่ติดตั้ง

Open via Lotus → ตรวจสุขภาพ ESP32... to diagnose and auto-fix the ESP32 toolchain.

It checks:

  • ESP32 core completeness — verifies tools/ not just hardware/.
  • gcc compiler — supports both esp-x32 (Xtensa) and esp-rv32 (RISC-V) of core 3.x.
  • Windows Defender exclusion — if the IDE folder isn't excluded, a Fix button auto-adds it via UAC.
  • Build cache — inspects %TEMP%/lotus_build/ and offers cleanup if stale.
⚠ Defender slow build Without exclusion, first compile on Windows can be many times slower — click Fix once after installation.

14. ติดตั้งไดรเวอร์ USB14. USB Drivers

เปิดจาก Tools → Install USB Drivers — รองรับสองชิป USB-Serial ยอดนิยม:

  • CP210x — ใช้กับ ESP32 DevKit, NodeMCU ส่วนใหญ่
  • CH340 / CH341 — ใช้กับ Arduino Nano clone, ESP8266 บางรุ่น
  • Open Drivers Folder — เปิดโฟลเดอร์เพื่อติดตั้ง manual

หลังติดตั้งไดรเวอร์ ต้อง ถอด USB แล้วเสียบใหม่ เพื่อให้ Windows assign COM port

Open via Tools → Install USB Drivers — supports the two common USB-Serial chips:

  • CP210x — most ESP32 DevKit and NodeMCU boards.
  • CH340 / CH341 — Arduino Nano clones, some ESP8266 boards.
  • Open Drivers Folder — opens the folder for manual installation.

After installing, unplug and re-plug the USB cable so Windows assigns a COM port.

อาการSymptom วิธีแก้Fix
ไม่เห็นพอร์ตในแอปNo port shown เช็คสาย USB (ต้องเป็นแบบ data, ไม่ใช่ charge only) · ติดตั้งไดรเวอร์ CP210x หรือ CH340 · ถอด-เสียบ USB ใหม่ Verify USB cable is a data cable (not charge-only) · install CP210x or CH340 driver · re-plug USB
คอมไพล์ ESP32 ช้ามากESP32 compile is very slow เปิด ตรวจสุขภาพ ESP32 → กด Fix เพื่อเพิ่ม Defender exclusion Run ESP32 Health Check → click Fix to add Defender exclusion
error: header not found ทั้งที่ไฟล์มีจริง"header not found" even when present cache เพี้ยน — ลบโฟลเดอร์ %TEMP%/lotus_build/ แล้วคอมไพล์ใหม่ Stale cache — delete %TEMP%/lotus_build/ and rebuild
Upload ล้มเหลว "Failed to connect" (ESP32)Upload fails: "Failed to connect" (ESP32) กดปุ่ม BOOT ค้างขณะกด Upload · ปิด Serial Monitor ก่อน · ตรวจว่าไม่มีโปรแกรมอื่นจับพอร์ตอยู่ Hold BOOT during upload · close Serial Monitor first · check no other app holds the port
Plugin ไม่ปรากฏใน ToolboxPlugin not in Toolbox รีสตาร์ท IDE หลังติดตั้ง · ตรวจที่แท็บ Installed ว่าติดตั้งจริง Restart the IDE after install · verify under the Installed tab
ESP32 core ติดตั้งค้างESP32 core stuck mid-install ปิดและเปิด IDE ใหม่ — ระบบจะตรวจพบและติดตั้งซ่อมอัตโนมัติ · หรือลบ core แล้วติดตั้งใหม่จาก Boards Setup Restart the IDE — it will auto-detect and repair · or uninstall core and reinstall via Boards Setup
GitHub login ล้มเหลวGitHub login fails ตรวจการเชื่อมต่ออินเทอร์เน็ต · ถ้ามี firewall ให้อนุญาต Lotus IDE · ลอง Sign out แล้ว Sign in ใหม่ Check internet connection · allow Lotus IDE through firewall · sign out and sign in again
